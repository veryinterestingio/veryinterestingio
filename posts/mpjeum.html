    Q:

    How does this little cheap device know every calculation I choose?

    finlshkd A:

    You know the rules for doing arithmetic operations on paper? Like "carry the one" and so on? Simply put, calculators pretty much just follow those same rules, but instead of carrying the one when they get to 10, they carry when they get to 2.

    Using 2, which is called "binary" is just more convenient because it can be represented by a bunch of indicators which are either on or off, which works out very nicely when it comes to electronics.

    Pretty much all the rules work out this way, and the same goes for carrying the one at pretty much any number. Although they're not the exact same, they work pretty similarly. As example, adding a 0 on the end doesn't multiply a number by 10, it multiplies by 2.

    Once the calculations have been completed in binary, the results can be converted to a base 10 system, the one we use in everyday life, and displayed on the screen. Likewise, the numbers you type in are converted to a binary, aka base 2, system before the computer really does anything with it.

    Quietm02 A:

    They calculate it. And I say that without trying to be funny.

    They don't "know" the answer. They work it out. To know the answer they would have to have every possible combination stored with its solution. Which just isn't feasible. (There may be a couple of specific examples like log tables or trig tables that are stored, but they're the exception).

    As for how it calculates, computers are really really good at doing sums. They can do them ridiculously fast. It will never make a mistake, so never needs to double check.

    There are small electrical components within the calculator that can do basic sums. Really fast. It uses them and does the calculation in stages until it gets the final answer. The same components can be used at multiple stages, as there's an overall programme that tracks where it is. And because it's so fast it looks instant to us.

    ComradeMicha A:

    All of the standard mathematical operations (add, subtract, multiply, divide) can be expressed as logical comparisons between two inputs, generating a specific output. To do it like the calculators do, you need to express all of the inputs and outputs as binary numbers. So "2" becomes "10", as in 1*(2\^1) + 0*(2\^0). 3 becomes "11", 4 becomes "100", 7 becomes "111" and so on. Now you can compare the individual 0s and 1s of both inputs and the output will be defined by a logical rule. For addition, you have the rule "not and", which means that the output is "0" if both inputs are the same, so if both have a "1" it becomes "0", if one is a "1" and the other is a "0" it becomes "1" and so on. Obviously there's another little trickery involved with carrying over a bit, but hey, this is ELI15 already ;)

    Example: 
2+3 becomes "10" + "11" in binary, so you compare from right to left:
0, 1 -> not the same -> result is "1"
1, 1 -> the same -> result is "0", plus carry over "1"
nothing (0), nothing (0), carry-over "1" -> not the same -> result is "1"
So the final result is "101", or 1*2^2 + 0*2^1 + 1*2^0 = 4+0+1 = 5.

    Now the only missing part of the explanation is how the binary conversion and comparison work. For the really cheap calculators, they actually used little nand-gates (microelectronics with two or three inputs and one output, where the logical comparison actually is done by the electrical setup of the gate itself. So if you put a certain voltage on the inputs (higher voltage is "1", almost no voltage is "0"), then they will have the resulting voltage on the output.

    Put enough of them together in a vast network, and you have a hardcoded addition machine. Now you just need a way to convert button inputs into voltage and voltage into display, both of which have been done for decades.

    Modern calculator just use all-purpose microprocessors which are programmed to calculate, using the same logic, but with a similar hardware than smartphones or other electronic devices.

    oneplusetoipi A:

    Not sure what part of the calculator you are most interested in, but here it goes.

    Parts of the calculator (the very simple cheap ones)

    • Battery to supply the power for the other stuff
    • Keyboard to input the numbers and the operator
    • Display to show the numbers (the ones you input and the ones that get output)
    • Chip (silicon based semiconductor integrated circuit) with: input connections, output connections, memory and computing elements. The chip has a small microprocessor for simple programming.

    Operation

    1. Enter the 1st number using the keyboard. The chip detects the key strokes and loads the numbers into the first memory location. The number is logically converted from decimal to binary during this stage.
    2. Enter the 'operator' using the keyboard (+, -, *, /). The chip is programmed to detect one of these keys as the end of the 1st number which locks the 1st number in memory. It also retains the knowledge of the operator for later.
    3. Enter the 2nd number using the keyboard. The chip detects the key strokes and loads the numbers into the second memory location. The number is logically converted from decimal to binary during this stage.
    4. Enter the '=' key using the keyboard. The chip is programmed to detect this key as the end of the 2nd number which locks the 2nd number in memory. The chip then proceeds to perform the operation associated with the operator entered earlier. (+, -, *, /)
    5. The chip now uses the program built in to the chip to select a pathway on the chip between the two memory locations and the ALU (Arithmetic Logic Unit). The ALU can add, subtract, and shift (shift is used for multiplying and dividing). For an ADD the two numbers enter the ALU and the circuits automatically compute the answer using a mechanism that is similar to what we do by hand. That is, each number that is the same 'power' is added and if there is a 'carry' that is passed to the next higher power part of the adder in the ALU. Remember the numbers are binary at this point so when you add values of the same power they are either a 1 or a 0 (result is 0, 1, or 2; and 2 needs a carry). Eventually all of the carry bits and adds complete in the adder and the new (binary) output can be saved to a third memory location. Note higher power adds need to include the carry so the result can be 0, 1, 2, or 3.
    6. Now the chip will use the value in the 3rd memory location to convert the binary result into a decimal number that will be on the display.

    I did not cover subtracting (uses the adder), multiplying or dividing, but I can if needed.

    ​

    Hope this helps.

    Binary for 0, 1, 2, 3 is 00, 01, 10, 11 respectively
