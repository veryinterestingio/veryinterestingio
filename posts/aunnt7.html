  • Mathematics why 1 by 49...499999 results in powers of 2?

    NEW
    Q:

    Hi, I was trying to understand why dividing 1 by 49 or 499... 499999 results in all the powers of 2 only varying by the 0 in between. Does 49 has any special quality in order to get those results? Thanks.

    1/49 1/499999

    DavidRFZ A:

    This is a fun quirk of the formula for the sum of a geometric series.

    Sum (a r^n ) as n -> infinity is a / (1 - r) when the absolute value of r is less than 1.

    Set a = 2/100 and r = 2/100 and you get (2/100) / (98/100) = 1/49

    or 1/49 = 2/10^2 + 4/10^4 + 8/10^6 + 16/10^8 + ....

    Set a = 2/1000 and r = 2/1000 and you get (2/1000)/(998/1000) = 1/499

    or 1/499 = 2/10^3 + 4/10^6 + 8/10^9 + 16/10^12 + ....

    Setting 'r' to a small multiple of 1/100 or 1/1000 or whatever makes the sums look really cool in decimal form.

    02474 A:

    I'll try to ELI5 a bit more simply than the other top-level comment.

    Dividing 1/49 is 0.02 and change simply because it is slightly larger than 1/50 (truly 0.02).

    Now, if you look at adding zeros to the denominator of the fraction 1/50, you'll get 1/500 (0.002), 1/5000 (0.0002), etc.

    Doing the exercise for 1/49, adding a 9 each time, is essentially doing the same exercise, but in each case, the values are merging closer and closer together in terms of absolute value. 1/49999 and 1/50000 are very much closer together compared to 1/49 to 1/50. Once you divide by enough powers of 10, 1/499....999 is, for all intents and purposes, the same as 1/500...000. For the same reasons why .999999...999 is mathematically equal to 1. https://en.wikipedia.org/wiki/0.999...

    MoobyTheGoldenSock A:

    I’ll try to do this with 4th-6th grade math instead of calculus.

    First let’s use whole numbers and work our way up to fractions. Let’s do 1000/8.

    Example 1

    We start with 1/8 in the thousands place. Nope, doesn’t work. Let’s try 10/8 in the hundreds place. That gives us a 1 in the hundreds, remainder 2 in the 100s place. We can write this remainder as a fraction, (2 x 10^2)/8.

    Dividing by 8 is the same as multiplying by 1/8, so we can write this remainder as 2 x 1/8 x 10^2. This means that 1000/8 = 100 + 2 x 1/8 x 10^2.

    We can pull a sneaky cheat here and say that 10^2 = 10 x 10. So we can write it as 2 x 1/8 x 10 x 10, and using associative property we can make it (2 x 10) x (1/8 x 10), or 20 x 10/8. We already know from before that 10/8 is 1 remainder 2, or 1 + 2/8. This means we can write 20 x 1/8 as 20 x (1 + 2/8). Using the distributive property, we get 20 + 40/8. This means that we can write 1000/8 as 100 + 20 + 40/8.

    40/8 is, of course, 5, so we can write 1000/8 = 100 + 20 + 5 or 125.

    Make sure you understand the above before reading on, and ask if you need clarification!

    Now let’s take an easy fraction, say, 1/8. I’ll go a bit quicker this time since it’s actually the exact same problem as 1000/8, just with the decimal in a different spot.

    Example 2

    1 doesn’t go into 8, so we add a decimal and go to 10ths. 10 divided by 8 is 1, remainder 2. Since 2 is in the 10ths column, it is actually 2 x 10^-1, and the next step would be to divide this result by 8. Thus, we can write 1/8 as 0.1 + (2 x 10^-1)/8.

    (2 x 10^-1)/8 is the same as 2 x 1/8 x 10^-1. We can sneaky cheat again and write that as 2 x 10^-1 x (0.1 + 2 x 10^-1). Using distributive property we get 0.2 x 10^-1 + 4/8 x 10^-2. So 1/8 is the same as 0.1 + 0.02 + 4/8 x 10^-2.

    4/8 is 1/2 or 0.5, so we can simplify 4/8 x 10^-2 to 0.5 x 10^-2, or 0.005. Thus, 1/8 = 0.1 + 0.02 + 0.005, or 0.125.

    Again, make sure you understand this before moving on.

    Example 3

    You may have noticed our sneaky cheat of turning all remainders into a number multiplied by 1/8 means we will always get a remainder of 2 when we divide 1 by 8, because 8 is 2 away from 10. This means we could also do the problem like this:

    1/8 = 10/8 x 10^-1 = 1 x 10^-1 + 2 x 10^-1 x 1/8

    So

    1/8 = 1 x 10^-1 + 2 x 10^-1 (1 x 10^-1 + 2 x 10^-1 x 1/8) = 1 x 10^-1 + 2 x 10^-2 + 4 x 10^-2 x 1/8

    So continuing the pattern, we get:

    1/8 = 1 x 10^-1 + 2 x 10^-2 + 4 x 10^-3 + 8 x 10^-3 x 1/8.

    A remainder 8 is problematic, because 8 is greater than or equal to 8 and we need our remainder to be smaller. Time to simplify:

    1/8 = 0.1 + 0.02 + 0.04 + 0.01 = 0.125.

    This version isn’t absolutely critical to understanding the problem, but it helps.

    Actual Problem

    So we do 1/49:

    1 doesn’t go into 49, so we add a decimal and go to 10ths. 10 doesn’t go into 49, so we add another decimal and go to 100ths. 100/49 is 2, so we have 0.02 (or 2 x 10^-2) with a remainder of 2 x 10^-2. As per earlier, we can write 1/49 as 2 x 10^-2 + 2 x 10^-2 x 1/49.

    Is getting late, so I’m going to throw caution to the wind and hope this isn’t too big a jump:

    We can take the 1/49 in the remainder another iteration, and we can use exponent notation instead of just writing 2 x 10^-2 over and over:

    1/49 = 2 x 10^-2 + (2 x 10^-2)^2 + (2 x 10^-2)^2 + 1/49

    And another iteration:

    1/49 = 2 x 10^-2 + (2 x 10^-2)^2 + (2 x 10^-2)^3 + (2 x 10^-2)^3 + 1/49

    And so on.

    When we convert this back to decimals, we’ll get:

    1/49 = 0.02 + 0.0004 + 0.000008 + 0.00000016 + 0.0000000032 + 0.000000000064 + 0.00000000000128 + ...

    And that’s... IDK, ELI14 maybe?
