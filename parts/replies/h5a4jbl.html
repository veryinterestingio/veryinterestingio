    • MrBeverly

      Compression allows you to say the same thing with less data by reducing redundancy.

      Consider the string "Greeeeeeeen" which takes 11 characters to store.

      I could compress it by storing the string as "Gre8n" which only takes 5 characters to store. Despite taking up less space, the same data is preserved.

      Now consider the string "Green" which already only takes 5 characters to store. If I compress it using the same "algorithm" it comes out as "Gre2n" which is still 5 characters. This string cannot be compressed further without losing data necessary to put it back together.

      Consider "Mississippi". If we tried to compress this, we may get "Mis2is2ip2i" which is still 11 characters. Despite all the repeating characters in the string, they're broken up by non-identical data which prevents compression from squeezing very effectively.

      When you compress a video, a solid blue sky would be the same thing as a string with a bunch of repeating characters next to each other. The compression algorithm can get a lot of mileage by replacing all the identical repeated data with one much shorter instruction. The red sky may have a lot more detail breaking up the solid, compressible areas which prevents you from getting a good result.

      Keep in mind the example provided above is very simplified and not what actual compression looks like, although it gets the general idea across.
