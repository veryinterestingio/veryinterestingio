    Yes, provided you're using uncommon words. With dictionary searches, you just expand the base set (e.g. try "aaa" through "aaz" and then "aaaardvark", "aaapple", "aabanana", etc. with common words equivalent to one brute forced character). So, while the words you chose are not very common, it wouldn't be very strong to have a password like "This is not going to be a very very good password." That's the same length as yours, but those are all very common words, so it's still relatively easy to brute force.

    Better would be unusual words, and even better still would be a mix of different languages ("avec fromage means with cheese").

    • cannons_for_days

      It depends highly on the algorithm used and whether or not the attacker is attacking an already-stolen list of password hashes or if they are attacking a server that has sane attempt policies in place. For most people, for most services, the only factors that really matter are that your password is unique (that is, you aren't using that same password for some other service), and that your password isn't just you putting some minor variation on whatever the most common pattern is.

      The only algorithm for which your example passphrase is "weak" is an algorithm that is trying to construct grammatically correct English sentences using the 100 most common English words, and then throws in one uncommon word ("password") somewhere. Under that algorithm, yes, the example sentence you gave is analogous to adding uncommon characters to a common password, or using a common pattern with an uncommon word. But nobody is using that algorithm to attack passwords because nobody is using that to create their passwords.

      (Also, I put "weak" in scare quotes up there because that's still a relatively strong password even against attackers using that algorithm. It's 2000x100^10. That's a 25-digit number. You can chop off five of those digits to represent the reduced complexity of having to conform to Proper English, but that still puts it in the 40th percentile. True, it's not strong enough to resist a targeted attack, but most people don't need to worry about that - they just need their password to take longer to break than the attacker lets their algorithm run. Which is usually "not all that long".)

      The real reason that using passphrases instead of passwords is useful is that is much easier to increase length and introduce complexity (by adding proper nouns, nicknames, or slang) without having the users forget it. The easier it is for the user to remember the password, the more likely they are to use a new one everywhere, and the more likely they are to change their passwords for important services more frequently than "never". Both of those are at least as important as making your password longer or increasing the dictionary size, but services are only capable of enforcing one of those, and they don't like it, so only security nerds ever really talk about it.
