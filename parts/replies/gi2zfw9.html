  • EightOhms

    This can be dangerous to have source code leaked because its easier for hackers to find holes in the security when they can see exactly how the system works.

    Which is why there is an argument that "open source" software can be more secure. When the source code is readily available to the public, the public can check it out and with more eyes on it, security holes can be found and fixed.

    With "closed source" where a company develops the software and keeps the source code private, there is always a chance the source code could leak and hackers can find weaknesses and the company isn't aware of it. Hackers can even find weaknesses without having the source code. But again, if the public can look at, for large important applications, like for example, voting machines, concerned software engineers can take a very critical eye to the source code and look for mistakes.
