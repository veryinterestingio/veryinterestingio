    • SaltwaterOtter

      Drivers/firmware are pieces of code designed to act as middlemen to the hardware. As such, they're given very privileged status when it comes to system security. Some exploits may leverage this "high security clearance" to mess with stuff ordinary software is not allowed to.

      • fubo

        Yep. Consider that many devices in your computer can read from and write to memory. (This is called DMA, or Direct Memory Access.) The way it's supposed to work is that the device and the OS agree on what part of memory will be used to communicate. But if the device or its driver were malicious, it could stomp on memory that was being used for a different purpose, like remembering your password for a website.
