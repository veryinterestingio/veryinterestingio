      • BTC_Brin

        Yes, having a lot of ram can be very helpful for certain data-heavy workloads: In a PC, the fastest memory is in the various caches on the CPU (L1 is fastest, L2 is slightly slower, etc.), the next fastest is the system RAM, and then the SSDs/HDDs are slower still. If you have enough RAM to put everything you’re working on in RAM, it will be much smoother and faster than if you don’t have enough RAM, and your system is constantly having to wait for data to load from SSD/HDD into RAM.

        At the moment, it seems that the practical limit for single-socket workstations is around 2TB. Working with raw high-res video footage (4K & 8K) is one potential use case where that much RAM could be very useful.

    One interesting thing to note is that because the address space is so large, we don't have to limit ourselves to just the lower part. Modern operating systems randomize where in the address space (but not in physical memory) programs and data are loaded to enhance security. This is known as Address space layout randomization.

    • TicTacMentheDouce

      And to add to this, this kind of "virtual" memory layout allows in some systems to use peripherals in your system (memory, graphics/dedicated hardware...) as simple internal memory, making it way easier to work with them.

      This is used in some ARM systems with 32bits available, but usually only a few megabytes(10^6) of actual memory. The rest of the 4GB available is used to access to peripherals.
