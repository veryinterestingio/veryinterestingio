    • KapteeniJ

      Randomness is property of thing generating those bits. If your process produces only bits 11111111, it's not very random.

      So to phrase it differently, if you get random bits in a bag, there is no compression algorithm which allows you to expect you are going to squeeze those bits to a smaller size. Only if you know that the bits aren't really random, that there is a structure to them, can you start compressing with expectation that you get something smaller than you started with.
