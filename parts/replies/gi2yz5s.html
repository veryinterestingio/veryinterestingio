    • Yancy_Farnesworth

      That output most likely doesn't include anything that happens inside of printf. Printf is a system call. Most of the time compiled programs will not include the machine code for system calls as those are delegated to the OS kernel. What that machine code probably includes is a bunch of operations to locate the memory address for the printf function and telling the CPU to start executing those instructions and where to find "Hello, World!" in memory. So you're definitely not getting the whole picture of what the CPU is doing to draw "Hello, World!" on your screen.

      There's a reason why abstraction is such a key concept in software. Something as simple as just making a function call actually involves dozens of CPU operations. Assembly programing is for masochists or the truly insane.
