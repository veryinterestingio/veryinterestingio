    Modern drives don't need explicit parking. Due to the way the motors are built, their neutral position is to be parked. They require active power to maintain position. The instant power is taken away, they return to neutral.

    What you may hear when power is removed is an audible click when the head returns uncontrolled to park.

  • marcan42

    Correctly written software should ensure that any data saved by the user is really on disk ("flushed") by the time the save operation ends, and also write data in such a way that it can never leave half-written data ("atomically"). One way to do this is to save the data to a new filename behind the scenes, then rename it over the actual file the user is working on. Either the old file is there, or the new one. There is no point in time where the data is lost or corrupted on disk.

    The problem is there is a lot of poorly written software out there that doesn't follow these rules properly. So while you can get away with hard shutdowns on modern hardware and modern OSes without issue, and while you won't have any trouble with software designed for reliability (e.g. any database system that is properly designed, like PostgreSQL, with safe/default configuration), if you make a habit of doing hard shutdowns, at some point you're going to run into shitty software doing a write operation unsafely and leaving a file corrupt.
