    The main thing an operating system allows us to do is run multiple programs at the same time.

    That might be kind of true now, but if you're taking a historical view of the subject, this is pretty far from true. Operating systems existed to abstract hardware access for a long time before thread management and multitasking came along. OSes true function is to handle hardware-level access so that software doesn't have to. If you're writing a spreadsheet application, you don't want to spend time handling interrupts for input or writing the code to format a disk and save a file to it. The OS says "just let me know when you need to make the speaker beep, and I'll take care of it" instead of having to know which CPU instruction to call in order to make that happen. That way, every single program isn't having to write tons of extra code just to make the computer work in addition to doing the thing the program is supposed to do.
