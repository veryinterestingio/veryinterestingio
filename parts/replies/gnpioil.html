    • Ericchen1248

      To add on to this.

      Most languages below the interpreted level go through a process called bootstrapping.

      So what this means is that when you were creating the C language, you don’t write all the tools to create C using assembly. You create a very basic set of C tools in assembly, then create the rest of the more complex tools with these new basic C tools, and maybe augment them with assembly if your previous tools don’t fully cover them. In most cases this tool would become the compiler

      Reason for this is because it is generally much quicker to implement something at a higher level from programmers, and it is also easier for future users to maintain/extend it. You can now be a C programmer to improve on the C compiler with no knowledge of assembly.

      So if you go online to search, the compiler for C# is written in C#, the compiler for Java is written in Java, etc.
