      They can, and they have for a very long time. What a 32bit machine can't do is natively address memory beyond 4GB, and some of that memory is going to be dedicated to the kernel.

      Most 32-bit processors could address more than 32 bits worth of memory, it just required some extra tricks to do so. A CPU feature called PAE (Physical Address Extension) allowed the use of up to 36 bits of physical memory address space by OSes that supported it, making the memory limit 64GB.

      • Captain-Griffen

        You don't have to fit a number into one segment. There's a performance hit, but it's incredibly rare* you need more than 64 bit precision. If for some reason you do, you can spread it over multiple sets of 32/64 bits for a performance overhead.

        *Encryption is one area where you do need more than that. Chips tend to have special support built in for that to make it quicker, since your computer does a lot of encryption/decryption.

        When Final Fantasy VII came out for the PlayStation, it didn't fit on a single disc. So they chunked the game into three pieces and put it on three discs. It creates a small hassle to have to deal with several discs just to play one game, but it's doable.

        When you have a number that doesn't fit into a 32 bit register, you do the same thing. Chunk it up into pieces and put the pieces in multiple registers. It creates a small hassle to have to deal with several registers just to use one value, but it's doable.
