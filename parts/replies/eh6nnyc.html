    • EncumberedOrange

      It's hard to write anything in less than a few hundred lines of code. You have to think of every conceivable thing that a program must do and provide handling for that.

      You want to be able to count scores? That requires keeping track of what the score is, where the score counter is, which font it uses, which color it uses, and so on.

      You want the user to click with his mouse? That requires checking if the mouse button has been pressed, but not double pressed and not right clicked, within each tile, but not if the tile is already occupied by the opponents piece and not if you're in the menu or the it's the opponents turn.

      All of the logic house keeping requires code to handle what's happening and make sure it can only happen if every requirement is filled. That takes up a lot of code.
