    • alcaizin

      Because, while that's a good explanation, it's not really that simple. You might have a game that was designed to interact with a particular operating system version, and makes assumptions about the functionality that the OS will make available to the game. Now let's say we're two major versions away. Operating systems tend to preserve some backwards compatibility, and things that work well won't necessarily be rewritten between versions. So maybe 90% of the functionality that the game expects to be available is there, but when it hits something that uses the 10% that's no longer compatible it crashes. Or maybe the game was designed to take advantage of a feature in order to improve performance, and will still run if it isn't there, but will run more slowly even on newer hardware.
