      • sooper_genius

        An example of this is error correction. When you send data, you can send extra bits that are calculated in such a way that you can recover the original information if it's garbled. This correction has limitations; for example, it might be able to correct from errors in two other bits but not three other bits.

        Sometimes errors are created in such a way as to make the error correction think it has fixed the problem, when it has not (because the error was way worse than the correction can account for).

        I had this problem when we lost a server and some files were restored from errored backup tape. Only about 3% of the files were "recovered", but they were still all garbage. The errors on the backup were worse than the corrections could compensate for.
